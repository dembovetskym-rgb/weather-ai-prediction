# Описание программы «Прогноз погоды (ML)»

## 1. Общая схема работы

Программа позволяет:
1. Создавать **профили моделей** (SSA или Бустинг) с параметрами обучения.
2. **Обучать** модель на исторических данных из Open-Meteo по выбранному городу и периоду.
3. **Прогнозировать** погоду на выбранную дату и горизонт (в часах).

Данные хранятся в `%LocalAppData%\WeatherAppML`: файл `profiles.json` и папки `models\{profileId}\` с файлами моделей (`.zip`) и для бустинга — файлами истории (`.history.json`).

---

## 2. Модели данных (примеры)

### ModelProfile — настройки одной модели

```csharp
// Ключевые поля профиля
public int WindowSize { get; set; } = 24;   // для SSA: окно в часах
public int SeriesLength { get; set; } = 48; // для SSA: длина ряда
public int TrainSize { get; set; } = 0;     // 0 = все данные
public int Horizon { get; set; } = 24;      // горизонт прогноза (часы)
public ModelType ModelType { get; set; }     // Ssa или Boosting
public DateTime? LastTrainEndUtc { get; set; } // конец обучения — от него считается старт прогноза
```

### WeatherDataPoint — одна почасовая точка из API

```csharp
public DateTime Time { get; set; }
public float TemperatureC { get; set; }
public float Humidity { get; set; }
public float Pressure { get; set; }
public float WindSpeed { get; set; }
```

---

## 3. API (WeatherApiService)

Используется **Open-Meteo** (без ключа).

- **Геокодинг:** по имени города возвращается первый результат с координатами и названием.

```csharp
var location = await _apiService.GetLocationAsync("Moscow", ct);
// location.Latitude, location.Longitude, location.Name
```

- **Архив:** по широте/долготе и диапазону дат запрашиваются почасовые данные (температура, влажность, давление, ветер).

```csharp
var data = await _apiService.GetHistoricalHourlyAsync(
    location.Latitude, location.Longitude, startUtc, endUtc, ct);
// data — список WeatherDataPoint по часам
```

---

## 4. Построение рядов (DatasetBuilder)

Из списка точек погоды делаются **четыре отдельных ряда** (по одной метрике).

```csharp
var series = _datasetBuilder.BuildSeries(orderedData);
// series[MetricType.TemperatureC] — List<float> температур по времени
// series[MetricType.Humidity], .Pressure, .WindSpeed — аналогично
```

Эти ряды передаются в обучение: для каждой метрики обучается своя модель.

---

## 5. Обучение SSA (TrainingService.TrainMetric)

Для **SSA** по одному ряду (например, температура):

1. Вычисляется `trainSize`: если в профиле `TrainSize == 0`, берётся весь ряд, иначе — указанное число точек.
2. Ряд загружается в ML.NET как последовательность `TimeSeriesData { Value }`.
3. Строится пайплайн **ForecastBySsa** с параметрами из профиля (WindowSize, SeriesLength, Horizon и т.д.).
4. Вызывается `Fit(dataView)`, модель сохраняется в `{metric}.zip`.

```csharp
var pipeline = _mlContext.Forecasting.ForecastBySsa(
    outputColumnName: nameof(ForecastOutput.Forecasted),
    inputColumnName: nameof(TimeSeriesData.Value),
    windowSize: profile.WindowSize,
    seriesLength: profile.SeriesLength,
    trainSize: trainSize,
    horizon: profile.Horizon,
    confidenceLevel: 0.95f,
    ...);
var model = pipeline.Fit(dataView);
_mlContext.Model.Save(model, dataView.Schema, modelPath);
```

Прогноз по SSA ограничен: модель выдаёт ровно `Horizon` шагов вперёд от конца обучения; сдвиг по времени задаётся параметром `startOffsetHours` при вызове прогноза.

---

## 6. Обучение бустинга (TrainingService.TrainBoostingMetric)

Для **бустинга** по одной метрике:

1. Для каждого индекса `i >= LagCount` строится строка:
   - **Признаки:** время (sin/cos часа, sin/cos дня года, день недели, месяц) + 24 лага (значения ряда в моменты `i-1 .. i-24`).
   - **Метка:** значение ряда в момент `i`.
2. Таблица загружается в ML.NET, обучается **FastTree** регрессия.
3. Модель сохраняется в `{metric}.zip`.
4. В отдельный файл `{metric}.history.json` записываются последние 24 значения ряда и min/max (для рекурсивного прогноза и клэмпинга).

Пример построения признаков (упрощённо):

```csharp
// Время → 6 признаков
features[pos++] = MathF.Sin(2*PI*time.Hour/24);
features[pos++] = MathF.Cos(2*PI*time.Hour/24);
features[pos++] = MathF.Sin(2*PI*time.DayOfYear/365);
features[pos++] = MathF.Cos(2*PI*time.DayOfYear/365);
features[pos++] = (int)time.DayOfWeek;
features[pos++] = time.Month;
// Лаги: 24 предыдущих значения
for (var lag = 1; lag <= 24; lag++)
    features[pos++] = series[index - lag];
```

## 7. Прогноз SSA (ForecastService)

1. По профилю проверяется `LastTrainEndUtc`; от него считается начало прогноза с учётом `startOffsetHours`.
2. Формируется список `ForecastPoint` с временами на нужный горизонт (с учётом ограничения SSA).
3. Для каждой метрики загружается сохранённая модель, создаётся **TimeSeriesEngine**, вызывается `Predict()` — получается массив на `Horizon` шагов.
4. В точки записываются значения из этого массива со сдвигом `startOffsetHours` (чтобы прогноз начинался с выбранной даты, если она в пределах горизонта).

```csharp
var engine = model.CreateTimeSeriesEngine<TimeSeriesData, ForecastOutput>(_mlContext);
var forecast = engine.Predict();
// forecast.Forecasted[i] — прогноз на (i+1)-й час от конца обучения
for (var i = 0; i < length; i++)
    applyValue(points[i], forecast.Forecasted[i + startOffsetHours]);
```

---

## 8. Прогноз бустинга (ForecastService.ForecastBoosting / ApplyBoostingMetric)

1. Из `{metric}.history.json` читаются последние 24 значения и время последней точки.
2. Нужно получить прогноз на `startOffsetHours + horizon` шагов. Цикл по шагам:
   - текущее время += 1 час;
   - из очереди лагов и времени строятся признаки (тот же формат, что при обучении);
   - модель предсказывает одно значение;
   - значение ограничивается диапазоном (ClampBoostingValue) и добавляется в очередь лагов (старый первый лаг удаляется).
3. Полученный ряд сглаживается скользящим средним (SmoothSeries).
4. В результирующие точки записывается срез, начиная с индекса `startOffsetHours`, на длину `points.Count`.

Таким образом, бустинг может «идти» на много шагов вперёд (в т.ч. на выбранную дату), но качество на дальних шагах падает; сглаживание и клэмпинг уменьшают скачки.

## 9. UI (Form1)

- **Вкладка «Профили»:** список профилей, выбор активного, добавление/удаление/сохранение, настройки (имя, тип модели, окно, ряд, train size, горизонт).
- **Вкладка «Обучение»:** город, период дат, кнопки «Обучить» / «Дообучить», таблица загруженных данных. Перед обучением загружаются данные через API, показываются в таблице, затем вызывается `TrainWithDataAsync`.
- **Вкладка «Прогноз»:** город (подсказка), дата прогноза, горизонт в часах, кнопка «Прогнозировать», таблица результата. По выбранной дате и горизонту вычисляются `startOffsetHours` и `horizon`, вызывается `ForecastService.Forecast(profile, startOffsetHours, horizon)`.

Для бустинга ограничение по горизонту не применяется (можно запрашивать много часов); для SSA прогноз обрезается по `profile.Horizon`.

## 10. Кратко по файлам

| Компонент | Назначение |
|-----------|------------|
| **WeatherApiService** | Геокодинг города и загрузка архивных почасовых данных Open-Meteo. |
| **ModelProfileService** | Чтение/запись profiles.json, CRUD профилей, пути к .zip и .history.json. |
| **DatasetBuilder** | Преобразование списка WeatherDataPoint в ряды по метрикам. |
| **TrainingService** | TrainAsync / TrainWithDataAsync: проверки, построение рядов, TrainMetric (SSA) или TrainBoostingMetric + SaveBoostingHistory. |
| **ForecastService** | Forecast: ветвление SSA/Boosting; SSA — Predict по каждой метрике; Boosting — рекурсивный шаг, клэмпинг, сглаживание. |
| **Form1** | Инициализация сервисов, привязка списка профилей и таблиц, обработка кнопок и валидация. |

Комментарии в коде помечены через `/// <summary>` и `//` в ключевых местах; в этом файле приведены примеры логики и фрагменты кода для понимания всей программы.
